# Stage 07 — Testing & Quality Assurance

## Purpose
Check everything before launch.

### Activities
- Mobile testing
- Browser testing
- Form testing
- Speed testing
- Accessibility review
- SEO checks

### Deliverables
- Bug fixes
- Final approval

## Inputs
- Working build from Stage 06-development

## Outputs
- Tested and approved website
- List of resolved issues

## Process
1. Test on multiple devices and browsers.
2. Test all forms and interactive elements.
3. Run performance / speed tests.
4. Review accessibility (contrast, keyboard nav, screen reader, etc.).
5. SEO checks (meta, headings, alt text, etc.).
6. Log and fix bugs.
7. Get final client sign-off.
8. Update `../../MEMORY.md` — mark complete and move to launch.

## Rules
- Hard gate: Do not launch until QA is complete and signed off.
- Document any deferred items.

## Checkpoint
All critical issues resolved. Final approval obtained.
