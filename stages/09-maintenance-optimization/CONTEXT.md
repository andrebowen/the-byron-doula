# Stage 09 — Maintenance & Optimization

## Purpose
Improve the site based on real user behaviour.

### Activities
- Content updates
- SEO improvements
- Performance monitoring
- Conversion optimization
- Security updates

## Inputs
- Live site analytics and user feedback
- Ongoing business goals

## Outputs
- Regular updates and improvements
- Performance reports
- Archived project materials if closing

## Process
1. Monitor analytics, Core Web Vitals, conversions.
2. Perform regular content updates and SEO tweaks.
3. Optimize based on real user data (A/B tests, heatmaps, etc.).
4. Apply security patches and dependency updates.
5. Schedule periodic reviews (e.g. quarterly).
6. When project winds down, archive deliverables and hand over docs.

## Notes
This is ongoing work after the initial launch.
