# Stage 02 — Wireframing

## Purpose
Create low-fidelity layouts that focus on structure rather than visual design.

### Activities
- Page layouts
- Information hierarchy
- Placement of content and calls-to-action
- User flow testing

### Deliverables
- Wireframes for key pages

Think of wireframes as the blueprint of a house—they show where things go, not what they look like.

Example:
```
--------------------------------
Logo

Headline
Subheadline

[ CTA Button ]

Image

Services
- Service 1
- Service 2
- Service 3

Contact Form
--------------------------------
```

## Inputs
- `stages/01-discovery-strategy/output/intake.md` (from Stage 01)
- Sitemap and content outline

## Outputs
- Lo-fi wireframes for each key page (home, about, services, contact, etc.)
- Mobile + desktop layouts
- Annotated hierarchy notes

## Process
1. List all pages from the sitemap.
2. For each page, define: primary headline, supporting sections, CTA placement, navigation.
3. Sketch structure as a lo-fi wireframe (Figma, FigJam, hand-drawn, or HTML lo-fi).
4. Annotate content priority (what the eye hits first, second, third).
5. Get client approval before moving to visual design.
6. Update `../../MEMORY.md` — mark Stage 02 complete and current stage → 03-visual-design.

## Rules
- Content drives layout — reference draft copy if available.
- No colors, no real fonts — wireframes are structural only.
- Mobile layout first, then desktop.
- Every page needs a primary CTA defined.
