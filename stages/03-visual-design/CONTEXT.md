# Stage 03 — Visual Design (UI Design)

## Purpose
Apply branding, colors, typography, imagery, and styling.

### Activities
- Brand integration
- Color palette
- Typography
- Component design
- Mobile and desktop layouts

### Deliverables
- High-fidelity mockups
- Design system or style guide

This is where the wireframe becomes something that resembles the final website.

## Inputs
- `stages/01-discovery-strategy/output/intake.md` (from Stage 01)
- Approved wireframes (from Stage 02-wireframing)
- Brand references and competitor research

## Outputs
- Design system: colors, typography, spacing scale, component library
- High-fidelity mockups for all key pages
- Mobile + desktop variants
- Component specs for build handoff

## Process
1. Define design system tokens (colors, type scale, spacing, border radius, shadows).
2. Design reusable components (buttons, cards, nav, footer).
3. Build out each page mockup using the design system.
4. Review mobile layout for every page.
5. Get client approval before moving to prototype.
6. Update `../../MEMORY.md` — mark Stage 03 complete and current stage → 04-prototype.

## Rules
- Use the design system — no one-off styles.
- Real copy in mockups, not lorem ipsum.
- Approval required before any code is written.

## Checkpoint
High-fidelity designs approved.
