# Stage 06 — Development

## Purpose
Build the website.

### Activities
- Front-end development
- CMS setup
- Forms and integrations
- Performance optimization
- Mobile responsiveness

### Deliverables
- Working website

## Inputs
- Approved designs, prototype, and final content (from previous stages)
- `stages/01-discovery-strategy/output/intake.md`

## Outputs
- Fully functional website in `../../../site/` (or equivalent)
- Any CMS content populated
- Integrations (forms, analytics, etc.) working

## Process
1. Set up project structure / repo / hosting preview.
2. Implement front-end from approved designs (HTML/CSS/JS, or framework/CMS).
3. Set up CMS if applicable and populate with approved content.
4. Implement forms, CTAs, and third-party integrations.
5. Optimize performance and ensure mobile responsiveness.
6. Hand off for testing.
7. Update `../../MEMORY.md` — mark Stage 06 complete and current stage → 07-testing-qa.

## Rules
- Build to the approved designs and content.
- Use the design system / tokens established earlier.
- No new features without approval.

## Checkpoint
Working site ready for QA testing.
