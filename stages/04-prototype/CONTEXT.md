# Stage 04 — Prototype

## Purpose
Create an interactive version of the design.

### Activities
- Clickable navigation
- Page transitions
- Form interactions
- Mobile responsiveness testing

### Deliverables
- Interactive prototype (often in Figma)

The prototype lets stakeholders experience the site before development begins.

## Inputs
- Approved high-fidelity mockups and design system (from Stage 03-visual-design)

## Outputs
- Clickable interactive prototype
- Annotations for interactions and states

## Process
1. Import or recreate designs in prototyping tool (Figma, etc.).
2. Add clickable navigation and links between pages.
3. Implement key interactions (forms, menus, modals, transitions).
4. Test on mobile and desktop viewports.
5. Share prototype for stakeholder review and feedback.
6. Get approval before moving to content finalisation or development.
7. Update `../../MEMORY.md` — mark Stage 04 complete and current stage → 05-content-creation.

## Rules
- Focus on user flow and interactions, not final code.
- Document any open questions or decisions needed.

## Checkpoint
Prototype approved by client. Ready for development handoff.
